Рекомендуемые платформы
Существуют управляемые сервисы, которые упрощают создание таких моделей:

DataRobot: Позволяет настроить политики переобучения с триггерами и действиями, например, переобучить модель, если 
производительность падает (DataRobot Automated Retraining).
AWS SageMaker: Использует Amazon SageMaker Model Monitor для обнаружения дрейфа данных и запуска конвейеров переобучения 
(AWS SageMaker Pipelines).
Google Cloud Vertex AI: Поддерживает конвейеры, которые можно автоматизировать на основе оповещений мониторинга или 
расписания (Vertex AI Training).
H2O Driverless AI: Предлагает автоматическое переобучение и обновление моделей на основе новых данных (H2O Driverless AI).
Azure Machine Learning: Обеспечивает автоматическое ML с возможностями мониторинга и переобучения (Azure Automated Machine 
Learning).
Для тех, кто предпочитает открытое ПО, можно рассмотреть MLflow или Kubeflow, но их настройка может потребовать больше 
технических знаний, что может увеличить вмешательство человека на начальном этапе.

Как выбрать платформу?
Рекомендуется изучить документацию и руководства по этим платформам, чтобы выбрать ту, которая лучше всего соответствует 
вашим потребностям и уровню технической экспертизы. Убедитесь, что платформа поддерживает загрузку ваших данных, так как 
вы упомянули, что хотите обучить модель на своих данных.

Отчет: Подробное исследование самообучающихся моделей с минимальным вмешательством человека
Этот отчет предоставляет детальное рассмотрение запроса на создание собственной самообучающейся модели, обученной на 
пользовательских данных с минимальным вмешательством человека. Мы исследуем концепции, технологии и платформы, которые 
могут удовлетворить этот запрос, включая управляемые сервисы и открытые решения.

Введение
Самообучающаяся модель в контексте машинного обучения (ML) — это система, которая может автоматически адаптироваться к 
новым данным, поддерживая или улучшая свою производительность без значительного участия человека. Это особенно важно в 
условиях, когда данные постоянно меняются, что может привести к дрейфу данных или концептуальным изменениям, влияющим на 
точность модели. Автоматизация процессов, таких как переобучение, является ключевым аспектом современной практики MLOps 
(операций машинного обучения), которая направлена на управление жизненным циклом моделей.

Понимание запроса
Пользователь выразил желание создать модель, которая будет обучена на их данных и способна самообучаться с минимальным 
вмешательством человека. Это подразумевает необходимость в системе, которая не только изначально обучается на предоставленных 
данных, но и может продолжать адаптироваться к новым данным без постоянного ручного управления. Это может включать 
автоматическое обнаружение необходимости переобучения, сбор новых данных, запуск процессов переобучения и обновление модели 
в производственной среде.

Ключевые концепции
Для достижения самообучения с минимальным вмешательством человека важно рассмотреть следующие аспекты:

Онлайн-обучение: Модели, которые могут обновляться инкрементально по мере поступления новых данных, без необходимости 
полного переобучения с нуля.
Автоматизированное переобучение: Процессы, где модель автоматически переобучается на основе расписания, триггеров (например, 
дрейф данных) или доступности новых данных.
Мониторинг производительности: Системы, которые отслеживают метрики модели (например, точность, F1-оценка) и запускают 
переобучение, если производительность падает ниже определенного порога.
MLOps: Практика, включающая автоматизацию конвейеров для управления данными, обучением, развертыванием и мониторингом моделей.
Анализ доступных решений
Существуют как управляемые сервисы, так и открытые платформы, которые могут поддерживать самообучающиеся модели с минимальным 
вмешательством человека. Рассмотрим их подробнее.

Управляемые сервисы (AutoML платформы)
Эти платформы предназначены для упрощения процесса создания и управления моделями, особенно для пользователей с ограниченной 
экспертизой в ML. Они часто включают автоматизированные процессы переобучения, что соответствует запросу пользователя.

DataRobot:
DataRobot предоставляет возможность настройки до пяти политик переобучения на каждое развертывание. Каждая политика включает 
триггер (например, расписание или условие производительности), стратегию моделирования, настройки моделирования и действие 
замены (например, продвижение новой модели, если она лучше). Это позволяет минимизировать вмешательство человека, так как 
система может автоматически запускать переобучение и уведомлять пользователя о результатах.
Подробности: DataRobot Automated Retraining.
Пример: Настройка политики через NextGen UI, просмотр истории запусков (успешных или неуспешных) с указанием времени начала, 
окончания и длительности.
AWS SageMaker:
Amazon SageMaker предлагает инструменты для автоматизации переобучения через Amazon SageMaker Model Monitor, который 
отслеживает дрейф данных и производительность модели. Переобучение может запускаться на фиксированных интервалах, при 
достижении определенного порога вариации модели или при наличии достаточного количества новых данных. Интеграция с 
AWS Step Functions, CloudTrail и CloudWatch позволяет автоматизировать запуск конвейеров на основе событий, таких как 
загрузка новых данных в S3.
Подробности: AWS SageMaker Pipelines.
Пример: Мониторинг статистики данных и запуск переобучения, если дрейф данных превышает порог.
Google Cloud Vertex AI:
Vertex AI, унифицированная платформа ML от Google Cloud, поддерживает автоматизированные конвейеры для переобучения. Хотя 
AutoML сам по себе может требовать ручного переобучения с новыми данными, Vertex AI позволяет настроить конвейеры, которые 
запускаются на основе оповещений мониторинга или расписания. Например, можно использовать Cloud Logging для захвата оповещений 
о падении производительности и запускать Cloud Function через PubSub для запуска конвейера переобучения.
Подробности: Vertex AI Training.
Пример: Настройка автоматического переобучения на основе дрейфа данных, описанного в статьях сообщества (Continuous training 
with BigQuery ML and Vertex AI).
H2O Driverless AI:
H2O Driverless AI предлагает функцию AutoML Retrain, которая позволяет настроить расписание для автоматического переобучения 
модели на новых данных. Это включает мониторинг производительности и обновление модели без значительного вмешательства 
пользователя, что делает его подходящим для запроса.
Подробности: H2O Driverless AI.
Пример: Автоматическое обновление модели на основе новых данных, загруженных в систему.
Azure Machine Learning:
Azure ML предоставляет автоматическое ML с возможностями мониторинга и переобучения через управляемые онлайн-эндпоинты. Можно 
настроить конвейеры для переобучения на основе триггеров, таких как падение метрик производительности или доступность новых 
данных. Это упрощает процесс для пользователей, не имеющих глубоких знаний в ML.
Подробности: Azure Automated Machine Learning.
Пример: Настройка автоматического переобучения на основе расписания или событий, таких как обновление набора данных.
Открытые решения
Для пользователей, предпочитающих не зависеть от конкретного облачного провайдера, существуют открытые платформы, такие как 
MLflow и Kubeflow, которые могут быть настроены для автоматизации процессов переобучения. Однако их настройка может 
потребовать больше технических знаний и ресурсов, что может увеличить вмешательство человека на начальном этапе.

MLflow: Поддерживает управление жизненным циклом ML, включая эксперименты, воспроизводимость и развертывание. Можно 
интегрировать с инструментами планирования, такими как Apache Airflow, для автоматизации переобучения.
Kubeflow: Платформа для ML на Kubernetes, поддерживающая конвейеры для автоматизации рабочих процессов, включая переобучение. 
Требует настройки инфраструктуры, что может быть сложным для новичков.

Лучшие практики для автоматизации
На основе доступной информации, включая руководство Arize AI, можно выделить следующие лучшие практики для автоматизированного 
переобучения:

Используйте метрики, основанные на производительности (например, точность, F1-оценка), для определения необходимости 
переобучения.
Настройте триггеры на основе дрейфа данных или концептуальных изменений, обнаруживаемых через мониторинг.
Разработайте стратегии продвижения новой модели, например, сравнение производительности с текущей моделью перед заменой.
Обеспечьте обратную связь в цикле переобучения, чтобы учитывать новые данные и их влияние на модель.
Эти практики поддерживаются платформами, такими как DataRobot и SageMaker, которые предоставляют инструменты для мониторинга 
и автоматизации.

Рекомендации для пользователя
Учитывая желание минимизировать вмешательство человека, рекомендуется рассмотреть управляемые сервисы, такие как DataRobot, 
H2O Driverless AI или Google Cloud Vertex AI, которые предлагают готовые решения для автоматического переобучения. Эти 
платформы поддерживают загрузку пользовательских данных, что соответствует вашему запросу. Если вы предпочитаете больше 
контроля или работаете в среде с ограничениями на использование облачных сервисов, можно рассмотреть MLflow или Kubeflow, 
но будьте готовы к большей вовлеченности на этапе настройки.

Для начала изучите документацию платформ, чтобы понять, как настроить автоматическое переобучение. Например, в DataRobot 
можно настроить политики через NextGen UI, а в SageMaker — использовать Model Monitor для обнаружения дрейфа.

Заключение
Создание самообучающейся модели с минимальным вмешательством человека возможно через использование современных AutoML 
платформ и инструментов MLOps. Управляемые сервисы, такие как DataRobot, AWS SageMaker, Google Cloud Vertex AI, 
H2O Driverless AI и Azure Machine Learning, предлагают готовые решения для автоматизации переобучения, что делает их 
подходящими для вашего запроса. Открытые решения, такие как MLflow и Kubeflow, также возможны, но требуют больше 
технических знаний. Рекомендуется выбрать платформу, соответствующую вашему уровню экспертизы и потребностям, и изучить 
соответствующую документацию для настройки.
